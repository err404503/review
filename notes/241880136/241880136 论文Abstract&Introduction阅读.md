## 一、论文贡献框架

### 问题

动态近似成员查询（Bloom Filter 类问题），但不知道最终会插入多少元素。

已知 n 时：空间 Θ(n log(1/ε))。未知 n 时：已有实际方案空间 Ω(n log n)，构造naive。

### 本文贡献

下界：至少 (1-o(1))n log(1/ε) + Ω(n log log n)

↓ 紧致匹配 (tight)

上界：至多 (1+o(1))n log(1/ε) + O(n log log n)

下界证明（第 3 章）：压缩论证。
上界证明（第 4–5 章）：两个构造。

## 二、定理 1.1 和 1.2

### 定理 1.1（下界）

任意针对未知大小集合 S、假阳性率 ε 的近似成员数据结构，在某 n > u^δ 次插入后，必须使用至少(1 − o(1)) n log(1/ε) + Ω(n log log n)位的空间。

第一项 (1−o(1))n log(1/ε) 是已知 n 时就需要的（和传统下界一致）
第二项 Ω(n log log n) 是"不知道 n"付出的额外代价

注：条件 n > u^δ 的含义：δ∈(0,1) 为任意小常数；n 与全集大小 u 之间呈多项式关系（如 n > √u 或 n > u^{0.01}）。排除 n 极小（如 n = O(log u)）的退化情况。

### 定理 1.2（上界，未去摊销）

构造1（热身）
| 属性 | 值 |
|---|---|
| 空间 | (1 + o(1)) n log(1/ε) + O(n log log n) |
| 假阳性率 | ≤ ε |
| 假阴性 | 无 |
| 插入时间 | 期望摊还 O(1) |
| 查询时间 | O(log n)|
| 已知 n | 否 |

构造2（精妙）
| 属性 | 值 |
|---|---|
| 空间 | (1 + o(1)) n log(1/ε) + O(n log log n) |
| 假阳性率 | ≤ ε + u^{−c} |
| 假阴性 | 无 |
| 插入时间 | 期望摊还 O(1) |
| 查询时间 | 最坏 O(1) |
| 已知 n | 否 |

构造2（去摊销优化后）
| 属性 | 值 |
|---|---|
| 空间 | O(n log(1/ε) + n log log n) |
| 假阳性率 | ≤ ε + u^{−c} |
| 假阴性 | 无 |
| 插入时间 | 最坏 O(1) |
| 查询时间 | 最坏 O(1) |
| 已知 n | 否 |
