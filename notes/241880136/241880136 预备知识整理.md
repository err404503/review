## 一、§2 Preliminaries

### 1. 记号约定

| 记号 | 含义 |
|------|------|
| [n] | 集合 {1, 2, ..., n} |
| x ← X | 按随机变量 X 的分布采样一个值 x |
| x ← S | 从有限集 S 上按均匀分布采样一个值 x |
| U | 大小为 u 的全集，所有元素来自 U |
| S ⊆ U | 待表示的集合，大小为 n |

### 2. 单位代价 Word RAM 模型

- 元素取自 U ，每个元素可存储在w = ⌈log u⌉位长的单个字中
- 以下操作均在 w 位操作数上以常数时间完成：
  - 加法、减法
  - 按位布尔运算（AND, OR, XOR, NOT）
  - 任意位数的左移和右移
  - 整数乘法
- 数据结构效率分析的标准模型

### 3. k 重独立哈希函数族

**标准定义（k-wise independent）**：

函数族 H = {h: U → V}，若对任意 k 个互异的输入 x_1, ..., x_k ∈ U 和任意 k 个目标输出 y_1, ..., y_k ∈ V，有：

$$Pr_{h \leftarrow H}[h(x_1)=y_1 \wedge ... \wedge h(x_k)=y_k] = \frac{1}{|V|^k}$$

含义：随机选出的哈希函数 h 把 k 个不同的输入映射到 k 个指定输出的概率，等于在所有 |V|^k 种可能的输出组合中均匀选取的概率。即 k 个输入的哈希值看起来完全独立且均匀。

**弱化版（k-wise δ-dependent）**：

若从 H 采样的 h 给出的分布 (h(x_1), ..., h(x_k)) 与 U^k 上的均匀分布之间的**统计距离**不超过 δ。

## 二、§1.2 Related Work

### 2. Counting Bloom Filter （就是Bloom Filter++）

- Bloom Filter 的删除变体：每个 bit 换成一个计数器。
- 插入：k 个位置的计数器各 +1
- 删除：k 个位置的计数器各 −1
- 代价：计数器需 log log n 位（防止溢出）→ 空间增大Ω(log log n) 因子
- Cohen & Matias 将额外开销降至 O(n) 位。

### 3. 基于字典的近似成员查询

**方法**：

1. 压缩： h: U → [m]，其中 m = n/ε
2. 存储：对每个插入元素 x，计算 h(x) 并存入Succinct Dictionary
3. 查询：检查 h(x) 是否在字典中

**属性**：

- 无假阴性。
- 假阳性率 ε，压缩时哈希碰撞导致的。
- Succinct Dict空间是(1+o(1))nlog(u/n)+O(n)，所以空间是(1+o(1))nlog(1/ε)+O(n)。

**有无删除的区别**：

- 无删除：只存互异哈希值（集合）
- 有删除：存哈希值多重集（归约到标准成员问题）

### 4. 在线 vs 离线空间分离（采用AI辅助理解）

- 静态场景：已经做到 n log(1/ε) + o(n) 位，几乎无额外开销
- 动态场景：一论文给出了 Ω(n / log(1/ε))的额外空间，但需要知道n。

### 5. 动态完美哈希与检索（采用AI辅助理解）

- 静态做法：完美哈希到[n] + 每个键在数组中存 log(1/ε) 位元素签名，nlog(1/ε) + O(n)。
- 动态做法：上述基础上，已有给出Θ(n log log n)额外开销。
