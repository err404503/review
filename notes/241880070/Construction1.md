## 1.总体目标

在不知道集合最终大小 n 的情况下，实现动态近似成员查询（支持插入、无假阴性、假阳性率 ≤ ε），并且空间复杂度接近最优。

## 2.设计数据结构Bi

- 使用 Raman and Rao [RR03] 的动态字典
- 选择一个哈希函数 h: U → [m]，其中m ≈ 2^i / ε_i，ε_i = Θ(ε / i²)
- 对于每个插入的元素x，计算h(x)，然后把h(x)放入动态字典当中
- 查询时：计算 h(x)，检查它是否在字典里。
- 假阳性率 ε_i，查询时间复杂度O(1)，整体空间复杂度(1 + o(1)) ⋅ 2^i⋅ log(1/ε_i)

## 3.整体框架
- 要插入的元素按指数分块

  - B_1：第 1~2 个元素（2¹ = 2）

  - B_2：第 3~6 个元素（2² = 4）

  - B_3：第 7~14 个元素（2³ = 8）

  - ...
- 分别维护 B_1, B_2, ..., B_k（k ≈ log n）
- 每个B_i分别独立工作，互不干扰

## 4.插入操作

- 假设将要插入的是第m个元素
- 通过2^{i}-2 < m <= 2^{i+1}-2 得到该元素应该插入哪个B_i
- 如果该元素是B_i的第1个元素，就新建一个 B_i
- 将这个元素插入相应的 B_i

## 5.查询操作

- 对当前所有已经存在的 B_1, B_2, ..., B_k 逐个调用其成员查询。

- 如果任意一个 B_i 返回 Yes，就直接返回 Yes

- 所有 B_i 都返回 No，才返回 No

## 6.空间复杂度分析

- 单个Bi整体空间复杂度(1 + o(1)) ⋅ 2^i⋅ log(1/ε_i) <= (1 + o(1)) ⋅ n ⋅ (log(1/ε_i)+ O(1)) = (1 + o(1)) ⋅ n ⋅(log(1/ε) + log (i ^2)+ O(1))
- 相加后得到总空间复杂度(1+o(1))nlog(1/ε) + O(nloglogn)

## 7.时间复杂度分析

- 插入：O(1) 大部分插入都是直接插入，新建一个 B_i的时间成本被摊销
- 查询：O(logn) 最坏情况需要进行O(logn) 次B_i内部查询
